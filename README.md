# BoxOTP

가볍고 미니멀한 **OTP(TOTP) 관리 Chrome 확장 프로그램**.
등록 · 백업 · 수정 · 클릭 한 번에 복사 + 자동 채우기를 한 자리에서.

```
┌──────────────────────┐
│  ▣  BoxOTP           │
├──────────────────────┤
│  ┌────────┐ ┌──────┐ │
│  │ Google │ │GitHub│ │   ← 클릭 한 번
│  │ alice  │ │ bob  │ │     → 복사 + 자동 채우기
│  └────────┘ └──────┘ │
│  ┌────────┐ ┌──────┐ │
│  │  AWS   │ │ Notion│ │
│  └────────┘ └──────┘ │
│                  [+] │
└──────────────────────┘
```

---

## 소개

BoxOTP는 Google Authenticator·Authy 같은 인증 앱을 **브라우저 안에서** 사용할 수 있게 해주는 가벼운 확장 프로그램입니다. 사이트마다 핸드폰을 꺼내 6자리 코드를 옮겨 적는 번거로움을 줄이고, 클릭 한 번에 코드를 입력란에 자동으로 채워 넣습니다.

- **로컬 저장만 사용** — 외부 서버로 전송하지 않습니다. `chrome.storage.local`에만 보관.
- **표준 호환** — RFC 6238 TOTP, RFC 4226 HMAC 기반. Google Authenticator로 그대로 이전 가능.
- **외부 의존성 최소화** — 분석/추적 코드 없음. CSP는 `script-src 'self'`로 잠겨 있음.
- **한국어 UI · 4가지 테마** — 다크(기본) / 라이트 / 세피아 / 미드나잇

## 설치

### 개발자 모드 로드
1. Github 우측상단 Download ZIP으로 다운로드
2. Chrome 주소창에 `chrome://extensions` 입력
3. 우측 상단 **개발자 모드** 토글 켜기
4. **압축해제된 확장 프로그램 로드** 클릭
5. 이 `BoxOTP-ChromeExtention` 폴더 선택

업데이트(코드 변경 후): 같은 페이지에서 BoxOTP 카드 우측 하단의 새로고침 아이콘 클릭.

## 사용 방법

### 1) OTP 등록

확장 아이콘을 클릭 → 우하단 **+** 버튼 → 다음 세 가지 방식 중 선택:

| 방식 | 설명 |
|---|---|
| **직접 입력** | 이름 + 발급자 + Base32 시크릿 키를 손으로 입력. 자리수/주기/알고리즘 변경 가능. |
| **otpauth:// URI** | 사이트가 제공하는 `otpauth://totp/...` 문자열을 통째로 붙여넣기. |
| **QR 이미지** | QR 캡처 이미지를 클릭/드래그로 업로드. 화면에 찍힌 QR을 즉시 인식. |
| **🆕 화면 드래그 캡처** | "현재 페이지에서 드래그로 캡처" 버튼 → 페이지에 오버레이가 떠서 QR 영역을 직접 드래그로 선택. macOS 스크린샷처럼. |

### 2) 코드 사용하기 (클릭 한 번)

- **타일을 클릭**하면 자동으로:
  1. 현재 OTP 코드를 클립보드에 복사
  2. 현재 탭에 OTP 입력란이 있으면 **자동으로 채우고 input/change 이벤트 발생** (React/Vue/Angular 사이트 호환)
- 결과는 상단 토스트로 알림: `채움 + 복사됨` / `채움` / `복사됨` / `복사됨 (채움 건너뜀)`
- **수정 키 + 클릭**으로 자동 채우기를 건너뛰고 복사만 수행할 수 있음 → [7) 복사만 수행](#7-복사만-수행-수정-키--클릭) 참고. 분할 입력란을 JS 로 그려서 채움이 한 칸에 몰리는 페이지 회피용.

지원하는 OTP 입력란 패턴:
- `autocomplete="one-time-code"` (표준)
- `name`/`id`/`aria-label`에 `otp`, `2fa`, `totp`, `mfa`, `verif`, `인증`, `보안코드` 포함
- `inputmode="numeric"` 또는 `type="tel"` + `maxlength` 6/7/8
- `placeholder`에 `OTP` 또는 `인증번호`
- **분할 입력란** (6개의 `maxlength=1` 인접 input — SMS 코드 스타일) 자동 분리 입력
- 현재 포커스된 input

### 3) 보기 모드 전환

헤더의 **⊞ / ☰** 버튼으로 두 모드 전환:

- **타일 모드** (기본) — 이름 위주로 컴팩트하게 그리드. 진행바는 카드 하단 가는 라인.
- **상세 모드** — 코드, 원형 타이머, 발급자가 한눈에 보이는 카드 리스트.

선택한 모드는 저장돼 다음 실행 시에도 유지.

### 4) 정렬 & 이름 변경

- **드래그 앤 드롭** — 타일이나 카드를 끌어다 원하는 위치로 옮기기 (검색 중에는 비활성)
- **타일 ⋮ 메뉴** (hover 시 보이거나 우클릭):
  - **이름 변경** — 라벨/발급자만 빠르게 수정
  - **상세 수정** — 시크릿·자리수·알고리즘까지 모두 수정
  - **↑ 위로 / ↓ 아래로** — 한 칸씩 이동
  - **삭제**

### 5) 검색

헤더의 🔍 아이콘 클릭 → 이름/발급자로 즉시 필터링.

### 6) 테마 변경

헤더 우측 **⋮** 메뉴 → **🎨 테마** 섹션에서 선택:

- **다크** (기본) · **라이트** · **세피아** · **미드나잇**

여러 테마를 빠르게 비교해볼 수 있도록 테마를 선택해도 메뉴는 닫히지 않습니다. 선택값은 저장돼 다음 실행 시에도 유지.

### 7) 복사만 수행 (수정 키 + 클릭)

분할 입력란을 JS 로 그리는 일부 페이지에서는 자동 채우기 값이 첫 칸에 몰리는 경우가 있습니다. 이런 페이지에서는 **지정한 키를 누른 채 OTP 타일을 클릭**하면 자동 채우기를 건너뛰고 복사만 수행할 수 있습니다.

설정 방법 — 헤더 ⋮ 메뉴 → **⌨️ 복사만 수행 (수정 키 + 클릭)**:

| 항목 | 동작 |
|---|---|
| **사용 안 함** | 항상 복사 + 자동 채우기. 일반 모드. |
| **키 설정... / 현재 키: …** | 클릭하면 키 입력 모달이 열림. **원하는 키를 그대로 누르면** 자동으로 인식. |

키 입력 모달은 다음 패턴을 모두 지원합니다:

- **비-수정 키** 단독(`C`, `Space`, `F8` 등) — 누르는 즉시 인식.
- **수정 키 단독/조합**(Shift, Alt+Shift, Cmd 등) — 마지막 수정 키에서 손을 떼는 순간 인식.
- **수정 키 + 일반 키 조합**(`Shift+C`, `Cmd+Alt+1` 등) — 일반 키 누르는 순간 즉시 인식.
- 다른 조합을 누르면 덮어쓰기. 저장 / 취소 버튼 또는 **ESC** 단독으로 모달 종료.

기본값: macOS 는 **Cmd + 클릭**, 그 외 OS 는 **Shift + 클릭**.

> 💡 일반 키 바인딩(예: `C` + 클릭)은 BoxOTP 팝업 안에서 키를 누른 상태로 타일을 클릭해야 동작합니다. 팝업 바깥에서 누른 키는 추적하지 않습니다.

## 백업 & 복원

헤더 우측 **⋮** 메뉴에서 접근:

| 항목 | 동작 |
|---|---|
| 📤 백업 (평문 JSON) | 사람이 읽을 수 있는 JSON 파일로 export. 안전한 환경에서만 사용. |
| 🔒 백업 (암호화 JSON) | 비밀번호를 받아 **AES-GCM-256 + PBKDF2-SHA256 (200,000회 반복)**으로 암호화된 JSON 생성. 분실 시 복원 불가. |
| 📱 Google OTP용 QR 백업 | `otpauth-migration://` 표준 포맷의 QR 코드를 화면에 표시. **Google OTP 앱 → 메뉴 → QR 코드 스캔**으로 그대로 이전 가능. 항목이 많으면 자동으로 여러 장의 QR로 분할. |
| 📥 복원 (JSON) | 평문/암호화 JSON 백업 파일에서 복원. 시크릿 기준 중복 자동 제외. |
| 📄 TXT 가져오기 (otpauth) | 한 줄에 `otpauth://` URI 1개씩 들어 있는 텍스트 파일에서 일괄 등록. `#`으로 시작하는 줄과 빈 줄은 주석으로 무시. |
| 🗑️ 전체 삭제 | 모든 항목 삭제 (확인 모달). |

## 기능 한눈에

| 기능 | 설명 |
|---|---|
| ✅ RFC 6238 TOTP | SHA-1/256/512, 6·7·8자리, 30/60초 주기. RFC 표준 테스트 벡터 5개 통과. |
| ✅ 클립보드 복사 | 클릭 한 번. 토스트 알림. |
| ✅ 자동 채우기 | 현재 탭의 OTP 입력란 감지 + React-friendly 값 주입. 분할 입력란도 자동. |
| ✅ 화면 드래그 캡처 | activeTab 권한으로 페이지 위에 오버레이 → 드래그 → jsQR 디코드. |
| ✅ QR 이미지 업로드 | 파일 클릭/드래그 앤 드롭. |
| ✅ otpauth URI 붙여넣기 | 단일 / 일괄(TXT). |
| ✅ JSON 백업 | 평문 + AES-GCM-256 암호화. |
| ✅ Google OTP 이전 | `otpauth-migration://` protobuf 인코딩 + 자동 다중 QR 분할. |
| ✅ 드래그 정렬 | HTML5 DnD, 순서 영구 저장. |
| ✅ 빠른 이름 변경 | 타일 ⋮ → 이름 변경. |
| ✅ 타일 / 상세 뷰 | 토글 + 상태 저장. |
| ✅ 검색 | 이름·발급자 필터. |
| ✅ 한글 UTF-8 | 라벨/발급자/QR 인식 모두 UTF-8. |
| ✅ 4가지 테마 | 다크 · 라이트 · 세피아 · 미드나잇. 메뉴에서 즉시 전환. |
| ✅ 복사만 수행 (수정 키) | 사용자 지정 키(+조합)로 자동 채우기를 건너뛰고 복사만 수행. |

## 권한 사용 내역

| 권한 | 용도 |
|---|---|
| `storage` | OTP 목록·뷰 모드·캡처 결과 임시 저장 (모두 로컬). |
| `clipboardWrite` | 코드 클립보드 복사. |
| `activeTab` | 사용자가 팝업을 연 순간만 현재 탭에 접근 (자동 채우기 + 드래그 캡처). |
| `scripting` | 자동 채우기·캡처 오버레이를 위한 스크립트 주입. |

호스트 권한(`host_permissions`)은 요청하지 않습니다. 백그라운드에서 사용자 페이지에 접근하지 않습니다.

## 보안 고려사항

- 모든 시크릿은 **이 기기의 Chrome 프로필** 안에만 저장됩니다 (`chrome.storage.local`). 외부 동기화·전송 없음.
- 평문 JSON 백업은 **시크릿을 그대로 포함**하므로 안전한 저장소에 보관하세요. 가능하면 암호화 백업을 권장.
- 암호화 백업은 PBKDF2-SHA256(200,000회) + AES-GCM-256. 비밀번호 분실 시 복원 불가.
- 확장 프로그램의 권한 범위에 따라, BoxOTP가 설치된 Chrome 프로필에 접근할 수 있는 다른 사람/프로그램은 시크릿을 볼 수 있습니다. Chrome 프로필 비밀번호·기기 잠금을 사용하세요.
- 자동 채우기는 일반적인 OTP 필드 패턴만 감지합니다. 비표준 폼에서 잘못된 필드에 값이 들어가면 즉시 그 필드를 비우세요.

## 알려진 제한

- `chrome://`, `edge://`, `about:`, 확장 프로그램 페이지 등에서는 자동 채우기·화면 캡처가 동작하지 않습니다 (브라우저 정책).
- 화면 캡처는 활성 탭의 가시 영역만 캡처합니다. 스크롤이 필요한 페이지면 QR이 보이는 위치로 먼저 스크롤해 주세요.
- Google Authenticator 표준은 6 또는 8자리만 지원합니다. 7자리 OTP를 Google OTP로 이전하면 6자리로 강제 변환됩니다.
- HOTP는 등록·이전만 부분 지원 (코드 생성은 TOTP만).
